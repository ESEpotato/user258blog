（3）个人云盘

如何内网穿透？

想要跟树莓派不在同一个组网的其他设备远程访问其挂载的ssd内容，做到内网穿透（公对私）即可；解决方案有虚拟组网/FRP等；

  • 虚拟组网

zerotier：零氪；低速；要下载客户端；

  • FRP，Fast Reserve Proxy反向代理

腾讯云/阿里云：买个呗；（没钱不买，有钱也不买；会玩了再买不迟）

樱花FRP：签到领流量；普通节点/青铜节点/白银节点；

注册登录账号并实名认证；HTTP(S)需要实名认证ICP备案；

https://www.natfrp.com/

创建启动器的帮助文档：https://doc.natfrp.com/launcher/usage.html#login

对内网穿透有了简单了解后，咱们继续个人网盘的事；

在树莓派入门使用手册（一）过后，咱们的树莓派微机能根据需求玩起来用起来了，而我根据实际情况（有台WD-SSD-1T-2024：￥500）和需求，打算整个简单但能玩的NAS服务器；

参考链接：https://www.zhihu.com/collection/964553545

我将朋友分享的资源存在某盘，有隐私安全、上传&下载速度很慢等问题，有问题就有需求，下面逐步解决需求；

4.3 NAS服务器
4.3.1 NAS是什么？做什么的？
NAS：Network Attached Storage网络附属存储，做下载机/个人云盘/片库管理/书库管理等；

4.3.2 部署NAS
软件方案：群辉/威联通/华硕/OpenMediaVault（OMV）等；

（1）部署OMV环境

1）换源

sudo vi /etc/apt/sources.list

注释之前的，拷贝下面的中科大和清华镜像源到文件中

deb http://mirrors.ustc.edu.cn/raspbian/raspbian/ buster main contrib non-free rpi
deb http://mirrors.tuna.tsinghua.edu.cn/raspbian/raspbian/ buster main contrib non-free rpi
deb-src http://mirrors.tuna.tsinghua.edu.cn/raspbian/raspbian/ buster main contrib non-free rpi
sudo vi /etc/apt/sources.list.d/raspi.list

注释之前的，拷贝下面的中科大和清华镜像源到文件中

deb http://mirrors.ustc.edu.cn/archive.raspberrypi.org/debian/ buster main ui
deb http://mirrors.tuna.tsinghua.edu.cn/raspberrypi/ buster main ui
deb-src http://mirrors.tuna.tsinghua.edu.cn/raspberrypi/ buster main ui

更新系统

sudo apt-get update && sudo apt-get upgrade -y
2）安装并登录OMV

从github仓库获取安装脚本；

获取链接：https://github.com/OpenMediaVault-Plugin-Developers/installScript

再手动安装

sudo chmod +x ??? && sudo install ???
最后登录OMV网页（默认admin/openmediavault）；电脑浏览器输入http://树莓派ip

3）设置OMV（可选操作）

常规设置里设置自动登出时间为1天；

基于磁盘（SSD）创建共享文件夹，开启smb实现共享；

4）基于OMV安装omvextras插件（可选操作）

从清华源获取omv-extras镜像插件；获取链接：https://mirrors.tuna.tsinghua.edu.cn/OpenMediaVault/openmediavault-plugin-developers/pool/main/o/openmediavault-omvextrasorg/

再手动上传并安装；

（2）下载机

软件方案：transmission保种，qbittorrent日常下载等；

1）挂载硬盘

参考如下内容，根据实际情况来设置

sudo umount /dev/sda1
sudo mkdir /mnt/ssd
sudo vi /etc/fstab

添加如下内容

/dev/sda1 /mnt/ssd exfat defaults 0 0

确认生效

sudo mount -a && lsblk -f
2）pull && deploy transmission

pull

sudo docker pull linuxserver/transmission:arm32v7-4.0.2

deploy

sudo docker run -d
–name=transmission
–net=host
-e PUID=1000
-e PGID=1000
-e TZ=Asia/Shanghai
-e USER=admin
-e PASS=passwd
-v /data/transmission/config:/config
-v /data/transmission/downloads:/downloads
-v /data/transmission/watch:/watch
-p 9091:9091
-p 51413:51413
-p 51413:51413/udp
–restart=always
linuxserver/transmission:arm32v7-4.0.2

#上述deploy缺少权限选项的配置以及PUID、PGID（id $user;选root）

docker logs transmission看到打印，补充后即可

s6-svscan: warning: unable to iopause: Operation not permitted
s6-svscan: warning: executing into .s6-svscan/crash
s6-svscan crashed. Killing everything and exiting.
s6-linux-init-hpr: fatal: unable to reboot(): Operation not permitted
sudo docker run –privileged –cap-add=SYS_ADMIN -d
–name=transmission
-e PUID=
-e PGID=
-e TZ=Asia/Shanghai
-e USER=admin
-e PASS=passwd
-v /data/transmission/config:/config
-v /media/WDssd/downloads:/downloads
-v /data/transmission/watch:/watch
-p 9091:9091
-p 51413:51413
-p 51413:51413/udp
–restart=always
linuxserver/transmission:arm32v7-4.0.2
3）pull && deploy qbittorrent

pull

sudo docker pull linuxserver/qbittorrent:arm32v7-4.5.3

deploy,默认帐号密码admin/adminadmin

sudo docker run –privileged –cap-add=SYS_ADMIN -d
–name=qbittorrent
-e PUID=
-e PGID=
-e TZ=Asia/Shanghai
-e WEBUI_PORT=8080
-e TORRENTING_PORT=6881
-p 8080:8080
-p 6881:6881
-p 6881:6881/udp
-v /data/qbittorrent/config:/config
-v /media/WDssd/downloads:/downloads
–restart=always
linuxserver/qbittorrent:arm32v7-4.5.3
ubuntu镜像作为测试种子：https://cn.ubuntu.com/download/alternative-downloads
